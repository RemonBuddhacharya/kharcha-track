The Crystal Ball

Forecasting predicts where your money is going. Anomaly detection catches when something looks wrong. Together, they give your expense tracker a sixth sense.

Two Superpowers

Think of these features like a weather service and a smoke alarm. The forecast looks at recent patterns and makes predictions. The anomaly detector does not predict — it reacts when something seems off.

🔮
Forecasting

Predicts next month’s spending using past patterns. Like a weather forecast — it looks at recent history and extrapolates forward.

🚨
Anomaly Detection

Flags expenses that look unusual compared to your normal spending. Like a smoke alarm — it does not predict fires, it reacts when smoke appears.

How Forecasting Works — Moving Average

The forecast engine uses a technique called a moving average. It looks at your last few months of spending and averages them to predict the next month.

Forecast.php — Core Algorithm 
$windowSize = 3;
$recentMonths = array_slice($monthlyTotals, -$windowSize);
$movingAvg = array_sum($recentMonths) / count($recentMonths);
$predictedAmount = round($movingAvg, 2);
$confidence = min(0.9, 0.6 + (count($monthlyTotals) * 0.02));
Plain English

Look at the last 3 months of spending...

Take the average of those 3 months...

That average becomes next month’s prediction...

The more months of data we have, the more confident we are (up to 90%)

1

2
Average: (5000 + 6000 + 5500) / 3 = ₹5,500
3
Prediction for next month: ₹5,500
4
Confidence: 78% (you have 9 months of history)

Why Moving Averages Work

This technique — averaging recent data points to smooth out noise — is called a moving average. Stock traders use it, weather services use it, and now your expense app uses it too. It is simple but surprisingly effective for short-term predictions.

How Anomaly Detection Works

The anomaly detector uses normalization to put all expenses on the same scale, then flags anything that strays too far from normal.

Anomaly.php — Normalization 
$amounts = $expenses->pluck('amount')->toArray();
$minAmount = min($amounts);
$maxAmount = max($amounts);
$amountRange = $maxAmount - $minAmount;

$normalizedAmounts = [];
foreach ($amounts as $amount) {
    $normalizedAmounts[] = $amountRange > 0 ? ($amount - $minAmount) / $amountRange : 0;
}
Plain English

Grab all expense amounts as a list...

Find the smallest and largest amounts...

Calculate the range between them...

Scale every amount to be between 0 and 1...

If the range is zero (all same amount), everything is 0

Anomaly.php — Scoring 
$meanAmount = array_sum($normalizedAmounts) / $numSamples;
$distanceFromMean = abs($amount - $meanAmount);
$score = min(1.0, $distanceFromMean * 3);
Plain English

Find the average of all normalized amounts...

How far is this expense from the average?

The further from average, the more suspicious (max anomaly score: 1.0)
